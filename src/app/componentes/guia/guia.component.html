Excel y CSV

Codificacion del CSV

Si se quiere abrir el archivo de salida en excel se encontrara que se importa mal, los caracteres especiales (acentos, dieresis, etc.) se ven rotos, con la codificacion html. Para corregir eso, en vez de abrir el archivo directamente, procederemos a abrir un archivo en blanco (recomiendo dejar uno de "verificación"). Una vez dentro, vamos a datos;

Una ves seleccionado, se abrira una ventana nueva para configurar la importacion. es la misma que la ventana para poner el CSV en filas, pero tendremos un selector nuevo, para elegir que codificacion tiene el archivo. Abrimos y elegimos Unicode UTF-8. Tambien podemos seleccionar la opcion de los encabezados, pero no es necesario. A partir de ahi, el resto es normal para un CSV, se elige metodo de separacion por comas y deberia quedar bien.

Web

Esta seccion maneja las publicaciones de la pagina Web. actualmente no hay conexion con la API de TiendaNube, por lo que se maneja con CSV. La app permite reflejar en los CVS de actualizacion:

  • Precio
  • Precio de oferta
  • Envio gratis o no
  • Visualizacion en plataforma
  • Codigo de barras

La app no modifica:

  • Stock. estos se arrastran de la web a la app cuando se carga el csv de tiendanube
  • Cambios en nombre. Se deben realizar dentro de la web. Si la URL cambia, la app generara otro publicacion nueva, quedando la anterior obsoleta (recordar borrar)
  • cambios en Codigo de Oxi. se deben hacer manualmente en ambas plataformas

IMPORTANTE si se realizan modificaciones en la plataforma web, recordar reflejarlas en la app, o se desincronizaran y traera problemas a futuro al exportar (ofertas diferentes, etc)

La app automaticamente pone en 0 los valores de productos Stihl, debido a su normativa de venta digital. NO ES UN ERROR DE LA APP

Los stock de las publicaciones se actualizan al pasarle el CSV de TiendaNube. Automaticamente detectara el stock actual publicado y lo actualiza en los datos de la app.
