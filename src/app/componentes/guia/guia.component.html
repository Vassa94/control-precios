Consideraciones iniciales

  • Todos los archivos de excel usados por la app son CSV. no admite xls o xlsx
  • Los CSV de salida son UTF-8. Esto dignifica que estan codificados para nuestro teclado. Si se desea abrir en excel aqui puede ver como
  • Si las tablas no cargan, es posible que no se este ejecutando el archivo JAVA. puede corregirlo asi
  • Si la tablas de productos no cargan, es probable que la computadora de Ignacio este apagada o sin red. Verificar
  • Si los codigos de barra se ven rotos, es porque no tienen los 13 digitos. corregir por el correcto o agregar 0 para que se corrija

Codificacion del CSV

Si se quiere abrir el archivo de salida en excel se encontrara que se importa mal, los caracteres especiales (acentos, dieresis, etc.) se ven rotos, con la codificacion html. Para corregir eso, en vez de abrir el archivo directamente, procederemos a abrir un archivo en blanco (recomiendo dejar uno de "verificaci√≥n"). Una vez dentro, vamos a datos;

Una ves seleccionado, se abrira una ventana nueva para configurar la importacion. es la misma que la ventana para poner el CSV en filas, pero tendremos un selector nuevo, para elegir que codificacion tiene el archivo. Abrimos y elegimos Unicode UTF-8. Tambien podemos seleccionar la opcion de los encabezados, pero no es necesario. A partir de ahi, el resto es normal para un CSV, se elige metodo de separacion por comas y deberia quedar bien.
